dw.experience.PageMgr
 ===========================
 EXPERIENCE BUILDER BETA API
 ===========================

Provides functionality for getting and rendering experience builder managed pages. The basic flow is to fetch a page by ID first via PageMgr.getPage(pageID) and then subsequently call to PageMgr.renderPage(page) with the recently retrieved page. This will trigger page rendering from a top level perspective, i.e. the page serves as entry point and root container of components. As a related page or component template will likely want to trigger rendering of nested components within its regions it can do this by first fetching the desired region by ID via Component/Page.getRegion(regionID) and then call to PageMgr.renderRegion(region) with the recently retrieved region (and optionally provide RegionRenderSettings for customized rendering of region and component wrapper elements).

Various attributes required for rendering in the corresponding template can be accessed with the accordant methods of Page and Component.

Any retrieval of pages, components and regions will return null in case the experience builder feature is turned off. Calling to the render methods would also yield no rendering result in case this feature is turned off. The experience builder feature is a beta feature only and must not be used if not explicitly taking part in the corresponding beta program.

renderRegion
static renderRegion(region : Region, regionRenderSettings : RegionRenderSettings) : void

Renders a region by triggering rendering of all visible components within this region. For each of these components first the process function is invoked, followed by invoking the view function for the purpose of writing the render result to the current response. The combined render output will be wrapped by an HMTL element.
In order to provide styling for these wrapper elements of the components and the region some render settings can optionally be provided, which basically allows to configure which kind of tag is used for the wrapper element and which attributes the wrapper element contains. A sample output could look like this if RegionRenderSettings are applied with customized tag names and attribtues for the region and component wrapper elements.

In order to go with the default settings for the wrapper elements see renderRegion(Region).

static renderRegion(region : Region) : void

Renders a region by triggering rendering of all visible components within this region. For each of these components first the process function is invoked, followed by invoking the view function for the purpose of writing the render result to the current response. The combined render output will be wrapped by an HMTL element with system default settings applied. The following sample shows how this would look like for a 'pictures' region that contains two components of type 'image'. 

 {@code 
 

     

         ...
     

     

         ..
     

 

 }

The system default for region render settings are:

  • tag_name : div
  • attributes : {"class":"experience-region experience-[REGION_ID]"}

The system default for component render settings are:

  • tag name : div
  • attributes : {"class":"experience-component experience-[COMPONENT_TYPE_ID]"}

In order to provide your own settings for the wrapper elements see renderRegion(Region, RegionRenderSettings).

Parameters:
region - the region that will be rendered
